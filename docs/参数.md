# CLI 参数说明

`sapr run` / `sapr_experiment.exe run` 可用参数一览。默认值来自 `SolverConfig` 与 `main.cpp`。

## 用法

```text
sapr run [--input input] [--output output] [--spacing 5] [--row-width 40]
         [--seed 1] [--sa-iterations 250] [--initial-temperature 5]
         [--cooling-rate 0.96] [--routing-layers 1]
         [--dump-routing-eval]
         [--debug-search]
         [--render-dpi 200] [--render-name name]
         [--dump-btree] [--render-btree-name name]
         [--no-dump-sa-btree] [--dump-sa-btree-json-only]
```

另有子命令：`sapr validate [--input input]`，只校验输入，不求解。

## 路径与 IO

| 参数 | 默认值 | 含义 |
|------|--------|------|
| `--input` | `input` | 输入目录，需含 `modules.txt`、`pins.txt`、`nets.txt`、`constraints.txt` |
| `--output` | `output` | 输出目录；写入 `placement.txt`、`routing.txt`、layout 图、`metrics.json`、`routing_debug.json` 等 |

## 求解参数

| 参数 | 默认值 | 含义 |
|------|--------|------|
| `--spacing` | `5` | B*-tree packing 时模块之间的基础间距（μm）；实际 gap 还会与 routing space feedback 取 max |
| `--row-width` | `40` | 单行最大宽度上限；超出部分计入 `row_width_penalty` |
| `--seed` | `1` | 随机种子，控制 SA 扰动的可复现性 |
| `--sa-iterations` | `250` | 模拟退火迭代次数；`<= 0` 时不做 SA，只评估初始解 |
| `--initial-temperature` | `5` | SA 初始温度 |
| `--cooling-rate` | `0.96` | 每轮温度衰减系数：`T ← T * cooling_rate` |
| `--routing-layers` | `1` | 允许金属层数（`1`/`3`/`7` 等，范围 `[1,7]`）。写入 `GridConfig.layer_count`，统一限制 A*、obstacle、terminal access 与 detailed reroute。`1`=仅 M1（默认，适合验证 placement/拓扑）；`3`=M1–M3（有限 via）；`7`=恢复旧的自由多层行为 |

## 调试与评估输出

| 参数 | 默认值 | 含义 |
|------|--------|------|
| `--dump-routing-eval` | 关闭 | 在标准输出额外打印 routing/DP/packing/feedback 等评估细节（phi cost、routing cost、DP 状态数等） |
| `--debug-search` | 关闭 | 打印 SA 搜索过程调试信息（扰动、代价、row overflow 等） |
| `--no-dump-sa-btree` | 关闭（即默认开启） | 关闭 SA 每轮候选 B*-tree/layout 导出；默认会在 `--output/btree/iter_XX/` 写出单轮 JSON、树结构图、带 net 树图和 layout 图 |
| `--dump-sa-btree-json-only` | 关闭 | 仍导出每轮 `iter_XX.json`、`placement.txt`、`routing.txt`，但**不渲染**每轮 PNG；适合长 SA（如 100 轮）。与 `--no-dump-sa-btree` 同时出现时以本开关为准（仍写文本） |

说明：`routing_debug.json` **不是** CLI 开关控制的；求解成功且有诊断数据时会自动写到 `--output` 目录。  
`metrics.json` 同样每次 `run` 成功后自动写出，与 layout PNG、`btree/` 同级，内容为终端打印的基础指标（area/wirelength 等）以及完整 `routing_evaluation` 摘要（含 warnings）。  
`--dump-routing-eval` 仅控制是否把同一份评估细节额外打印到标准输出。  
默认开启的 SA 每轮 btree 可视化会导出每一轮扰动后的候选解 `next`（无论接受/拒绝）。每轮位于 `--output/btree/iter_XX/`，包含 `tree.png`（仅树结构）、`tree_with_nets.png`（树结构 + 全部 net 拓扑弧）、`iter_XX_layout.png`、`placement.txt`、`routing.txt` 和 `iter_XX.json`；图上标注扰动序号、move、changed、accept 与代价信息；`sa-iterations <= 0` 时不会生成该目录。  
`--dump-sa-btree-json-only` 时每轮目录只保留 JSON 与 placement/routing 文本，最终解的 layout PNG / `metrics.json` / `routing_debug.json` 仍会生成。  
SA 运行时默认向 stderr 打印轻量进度，例如 `[sa] 3/20 move=module-swap accept=true next_cost=... current_cost=... best_cost=...`；更详细的搜索诊断仍用 `--debug-search`。

## 渲染参数

| 参数 | 默认值 | 含义 |
|------|--------|------|
| `--render-dpi` | `200` | layout / B* 树 PNG 的 DPI，必须为正整数 |
| `--render-name` | （空） | layout 图文件名（不含扩展名）；为空时用渲染脚本默认名 |
| `--dump-btree` | 开启（兼容保留） | 写出最终解 `btree_trace.json` 并渲染 B* 树图；现已默认开启 |
| `--no-dump-btree` | 关闭 | 关闭最终解 B* 树导出 |
| `--render-btree-name` | （空） | B* 树图文件名（不含扩展名）；为空时用渲染脚本默认名 |

## 示例

```powershell
.\build\sapr.exe run `
  --input input `
  --output output `
  --spacing 5 `
  --row-width 40 `
  --seed 1 `
  --sa-iterations 250 `
  --initial-temperature 5 `
  --cooling-rate 0.96 `
  --routing-layers 1 `
  --dump-routing-eval `
  --dump-btree `
  --render-name layout `
  --render-btree-name btree
```

恢复旧多层行为：

```powershell
.\build\sapr.exe run --input input --output output --routing-layers 7
```

有限 via（M1–M3）ablation：

```powershell
.\build\sapr.exe run --input input --output output --routing-layers 3
```

长 SA 只留每轮 JSON（不画每轮图）：

```powershell
.\build\sapr.exe run `
  --input cases\4ring_2_left6_no_symmetry\input `
  --output cases\4ring_2_left6_no_symmetry\output_sa100_layers1 `
  --sa-iterations 100 `
  --routing-layers 1 `
  --dump-sa-btree-json-only `
  --dump-routing-eval
```
