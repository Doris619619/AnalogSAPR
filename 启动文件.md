# IO_example 启动命令记录

本文记录在 `IO_example` 仓库中使用 `input` 目录跑一次求解并生成 `output` 的命令。

## 1. 进入项目根目录

```powershell
cd "D:\梁彦诗的项目\Liang的科研\科研六iclr\IO_example\IO_example"
```

## 2. 构建项目

如果还没有 `build\sapr.exe`，先执行：

```powershell
cmake -S . -B build -G Ninja -DCMAKE_BUILD_TYPE=Release
cmake --build build
```

构建完成后，可执行文件位于：

```text
build\sapr.exe
```

## 3. 校验 input（可选）

```powershell
.\build\sapr.exe validate --input input
```

成功时会输出：

```text
OK
```

## 4. 正式运行

最小完整命令：

```powershell
.\build\sapr.exe run --input input --output output
```

因为 `--input` 和 `--output` 默认就是 `input` / `output`，也可以简写为：

```powershell
.\build\sapr.exe run
```

运行后会生成：

- `output/placement.txt`
- `output/routing.txt`
- `output/output_layout.png`

终端会打印面积、线长、bend、via、penalty 等指标。

## 5. 可选参数

```powershell
.\build\sapr.exe run `
  --input input `
  --output output `
  --spacing 5 `
  --row-width 40 `
  --seed 1 `
  --sa-iterations 250 `
  --initial-temperature 5 `
  --cooling-rate 0.96
```

需要查看 routing/packing 调试信息时：

```powershell
.\build\sapr.exe run --input input --output output --dump-routing-eval
```

如果只想生成 `placement.txt` 和 `routing.txt`，不生成 PNG：

```powershell
.\build\sapr.exe run --input input --output output --no-render
```

## 说明

`input` 目录需要包含：

- `modules.txt`
- `pins.txt`
- `nets.txt`
- `constraints.txt`

`input/IO_example.code-workspace` 是 VS Code 工作区文件，不参与求解。
